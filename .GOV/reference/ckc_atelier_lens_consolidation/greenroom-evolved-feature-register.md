# CKC Evolved Feature Register

CKC features created from real use and convenience pressure are requirement evidence for Atelier/Lens consolidation. They must not be ignored because they were built outside Handshake or arrived later than the original Atelier/Lens stubs.

SQLite is rejected in all Handshake contexts: runtime, tests, fixtures, mocks, examples, fallbacks, cache, compatibility adapters, temporary harnesses, imports, exports, and product paths.

| ID | Feature | Source Evidence | Why It Matters | Consolidation Decision | Notes / Guardrails |
|---|---|---|---|---|---|
| EVOL-001 | Stable public character IDs separate from internal IDs | `app/backend/db.js` `Character.public_id`; `app/backend/library.js` public-id helpers; tests `backend_public_character_id.test.js`. | Operator-facing character identity needs stable labels without leaking storage keys. | Fold. | Preserve as system-managed; no model may rewrite it as ordinary sheet text. |
| EVOL-002 | Typed character sheet parser with union and block-list fields | `templateParser.js`, `sheet.js`, `validation.js`, `BlockListEditor.tsx`, tests `template_parser_unions.test.js`, `validation_field_types.test.js`, `block_list_validation.test.js`. | CKC solved real sheet editing pain: typed fields, descriptor fallbacks, score normalization, nested block editing. | Fold. | Port semantics, not JS implementation; keep adult/explicit fields first-class and text-preserving. |
