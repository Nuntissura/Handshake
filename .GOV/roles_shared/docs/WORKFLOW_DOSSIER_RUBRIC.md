# Workflow Dossier Rubric

Use this rubric as the closeout judgment layer inside the Workflow Dossier.

## Purpose

This rubric scores:

1. Workflow Smoothness
2. Master Spec Gap Reduction
3. Token Cost Pressure
4. Communication Maturity
5. Terminal and Session Hygiene

It also requires explicit scanning for:

1. Silent failures and false greens
2. Wrong tool or command-family usage
3. Task, path, or worktree ambiguity
4. Read amplification and governance-document churn

## Migration Rule

During the migration window, the scoring model and probe families remain identical to `.GOV/roles_shared/docs/POST_SMOKETEST_IMPROVEMENT_RUBRIC.md`.

Use that detailed scoring guidance with these naming rules:

- the canonical section title is `## Workflow Dossier Closeout Rubric`
- `## Post-Smoketest Improvement Rubric` remains an accepted compatibility heading for older runs
- the ambiguity section remains mandatory

## Required Output Shape

Use these canonical subsection titles:

- `### Workflow Smoothness`
- `### Master Spec Gap Reduction`
- `### Token Cost Pressure`
- `### Communication Maturity`
- `### Terminal and Session Hygiene`

Each target must include:

- `TREND: IMPROVED | FLAT | REGRESSED`
- `CURRENT_STATE: LOW | MEDIUM | HIGH`
- `NUMERIC_SCORE: 0-10`
- `Evidence:`
- `What improved:`
- `What still hurts:`
- `Next structural fix:`

## Mandatory Drift Lens

Because the Workflow Dossier keeps mechanical evidence first, the closeout rubric must also assess:

- context drift between original session intention and final narrative
- cognitive drift between actual runtime movement and Orchestrator interpretation
- hallucinated or weakly supported claims in the final opinion layer

If drift is visible, name it explicitly and anchor it to ACP/runtime/receipt evidence.

## Compatibility

If a tool or document still points to the post-smoketest rubric, use the same scoring model but treat this file as the canonical naming and closeout intent.
